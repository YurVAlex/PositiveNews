Project Title: "PositiveNews".

Level: Educational.
Status: Development.

Description: 
A web platform combats "doom-scrolling" by aggregating news from various sources, 
store them, and ranking them based on calculated score. Provides UI for viewing news sorted 
by positivity

The "PositiveNews" features List:
1. Ingestion Engine: Receiving news from various sources.
2. Content Distiller: Get the core news text and short 1min-read summary.
3. Secure Storage: Storing articles, summaries, and metadata in a structured database.
4. Sentiment Assessor: Scoring articles for "positivity" (0.0 to 1.0).
5. Configurable Feed: Public visibility of with a global "positivity" threshold.
6. User Personalization: Authorized users can set their own positivity metrics.
7. Adaptive Reader: Toggle between "Summary" and "Full Article" views.
8. Community Interaction: Leaving and viewing comments on articles.
9. Admin Command Center: Operations for article, sources list editing and comment moderation.
10. Source Filtering: Narrowing the feed to specific trusted publishers (e.g., "Only Reuters").
11. Topic Filtering: Categorizing news by domain (e.g., "Technology," "Health").
12. Topics Sorting: By publication date, or "positivity" score.
13. Automatic logging: Logging of operations with writing to stored text files.
